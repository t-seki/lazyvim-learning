## 🎪 LazyVimプロジェクト管理とは？

**LazyVim = 統合プロジェクト管理環境**

### 従来のvim vs LazyVimプロジェクト管理

| 従来のvim | LazyVim |
|-----------|---------|
| 手動ディレクトリ変更 | **project.nvimによる自動検出** |
| `:mksession`コマンド | **persistence.nvimによる自動セッション** |
| netrw（基本的ファイラー） | **Neo-treeによる高機能ファイラー** |
| 手動バッファ管理 | **Telescopeによる統合バッファ管理** |

### LazyVimプロジェクト管理の階層

1. **📁 プロジェクトレベル**: 複数プロジェクトの管理
2. **📂 ディレクトリレベル**: フォルダ構造のナビゲーション
3. **📄 ファイルレベル**: バッファとウィンドウの管理
4. **🔧 セッションレベル**: 作業状態の保存・復元

## 🗝️ 重要なプロジェクト管理キーマップ

### プロジェクト・ファイル管理

| キー | 機能 | 重要度 |
|------|------|-------|
| `<leader>e` | Explorer Snacks | 高 |
| `<leader>fp` | Projects | 高 |
| `<leader>ff` | Find Files (Root Dir) | 高 |
| `<leader>fb` | Buffers | 中 |

### セッション・ワークスペース管理

#### セッション操作
| キー | 機能 | 説明 |
|------|------|------|
| `<leader>qs` | Restore Session | セッション復元 |
| `<leader>ql` | Restore Last Session | 最後のセッション復元 |
| `<leader>qd` | Don't Save Current Session | セッション保存停止 |

#### ウィンドウ・タブ管理
| キー | 機能 | 説明 |
|------|------|------|
| `<leader>w` | Window commands | ウィンドウ操作 |
| `<leader><tab>` | Tab commands | タブ操作 |
| `<C-h/j/k/l>` | ウィンドウ移動 | 左/下/上/右のウィンドウへ |

### ターミナル・外部ツール統合

| キー | 機能 | 説明 |
|------|------|------|
| `<leader>tf` | Float Terminal | フローティングターミナル |
| `<leader>th` | Horizontal Terminal | 水平分割ターミナル |
| `<leader>tv` | Vertical Terminal | 垂直分割ターミナル |
| `<leader>gg` | GitUi (Root Dir) | LazyGit起動 |

## 🎨 LazyVimプロジェクト環境の理解

### プロジェクト管理システムの連携

```
┌─────────────────────────────────────┐
│           ワークスペース              │ ← プロジェクト全体管理
├─────────────┬───────────────────────┤
│   Neo-tree  │     メインエディタ     │
│  ──────────  │  ──────────────────  │
│  📁 project  │  ┌─────────────────┐ │ ← ファイル内容
│  📂 src/     │  │ main.js         │ │
│  📄 main.js  │  │ function app()  │ │
│  📄 config/  │  └─────────────────┘ │
├─────────────┼───────────────────────┤
│  ターミナル   │     セッション状態     │
│  ──────────  │  ──────────────────  │
│  $ npm run   │  💾 自動保存         │ ← persistence.nvim
│  $ git log   │  🔄 状態復元         │
└─────────────┴───────────────────────┘
```

### 重要なUI要素

- **Neo-tree**: 統合ファイルエクスプローラー（Git状態・診断表示）
- **Project.nvim**: プロジェクト自動検出・切り替え
- **Persistence**: セッション自動保存・復元
- **ToggleTerm**: 統合ターミナル管理

## ⚡ 実践的なプロジェクト管理ワークフロー

### 開発開始時のフロー

1. **プロジェクト切り替え**: `<leader>fp` でプロジェクト選択
2. **セッション復元**: `<leader>qs` で前回の作業状態復元
3. **ファイル確認**: `<leader>e` でプロジェクト構造把握
4. **作業開始**: `<leader>ff` で作業対象ファイル検索

### 日常開発での管理フロー

- **ファイル間移動**: `<leader>fb` でバッファ間高速移動
- **ディレクトリ作業**: Neo-treeでファイル作成・削除・移動
- **ターミナル連携**: `<leader>tf` でコマンド実行
- **Git操作**: `<leader>gg` でバージョン管理
